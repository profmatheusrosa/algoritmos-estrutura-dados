# Glossário: Algoritmos e Estrutura de Dados

Este glossário contém definições concisas para termos técnicos utilizados ao longo do curso.

## A
- **Algoritmo:** Uma sequência finita de instruções bem definidas para resolver um problema.
- **Array:** Uma estrutura de dados que armazena elementos do mesmo tipo em memória contígua.
- **Árvore (Tree):** Estrutura de dados hierárquica com uma raiz e subárvores de filhos.
- **Assíntota:** Uma linha que uma curva se aproxima infinitamente. Usada na análise assintótica (Big-O).

## B
- **Backtracking:** Técnica algorítmica para encontrar soluções por tentativa e erro, "voltando atrás" quando um caminho falha.
- **Big-O Notation:** Notação matemática que descreve o limite superior da complexidade de um algoritmo.
- **Binary Search Tree (BST):** Árvore binária onde a subárvore esquerda tem valores menores e a direita valores maiores que a raiz.

## C
- **Cache:** Camada de armazenamento de alta velocidade para dados acessados frequentemente.
- **Colisão (Hash):** Quando duas chaves diferentes produzem o mesmo índice em uma tabela hash.
- **Complexidade de Espaço:** Quantidade de memória que um algoritmo precisa em relação à entrada.
- **Complexidade de Tempo:** Quantidade de tempo (passos) que um algoritmo leva em relação à entrada.

## D
- **Dijkstra:** Algoritmo para encontrar o caminho mais curto em grafos ponderados positivos.
- **Dynamic Programming (DP):** Método para resolver problemas complexos quebrando-os em subproblemas sobrepostos mais simples.

## F
- **Fila (Queue):** Estrutura FIFO (First-In, First-Out). O primeiro a entrar é o primeiro a sair.
- **Função Hash:** Função que mapeia dados de tamanho arbitrário para valores de tamanho fixo.

## G
- **Grafo:** Estrutura composta por vértices (nós) e arestas (conexões).
- **Guloso (Greedy):** Algoritmo que faz a escolha localmente ótima em cada etapa, esperando encontrar o ótimo global.

## H
- **Heap:** Árvore binária especializada que satisfaz a propriedade de heap (pai sempre maior ou menor que filhos). Usada em filas de prioridade.

## L
- **Lista Encadeada:** Coleção linear de elementos onde cada elemento aponta para o próximo.

## M
- **Memoization:** Técnica de otimização que armazena resultados de chamadas de função custosas.

## P
- **Pilha (Stack):** Estrutura LIFO (Last-In, First-Out). O último a entrar é o primeiro a sair.

## R
- **Recursão:** Processo onde uma função chama a si mesma direta ou indiretamente.
